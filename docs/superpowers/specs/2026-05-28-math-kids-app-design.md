# Math Kids App — Design Spec
**Дата:** 2026-05-28  
**Автор:** Denis Kukobin  
**Статус:** Approved

---

## Обзор

Мобильное приложение для детей 6–7 лет (1-й класс), обучающее арифметике на основе советского учебника математики 1956 года. Контент строго следует оригинальному учебнику. Приложение бесплатное, работает офлайн, публикуется в РуСторе.

---

## Цели

- Дать ребёнку структурированный путь через программу 1-го класса
- Сделать процесс вовлекающим за счёт базовой геймификации (звёзды, прогресс)
- Обеспечить работу без интернета и без регистрации
- Минимальный технический порог для разработки (один разработчик + AI-ассистент)

---

## Технический стек

| Компонент       | Решение                          |
|----------------|----------------------------------|
| Фреймворк      | React Native + Expo (Managed)    |
| Навигация      | Expo Router (file-based)         |
| Состояние      | Zustand                          |
| Хранилище      | AsyncStorage (локально на устройстве) |
| Язык           | JavaScript (TypeScript опционально) |
| Сборка         | EAS Build (облако Expo)          |
| Публикация     | РуСтор (Android APK/AAB)         |
| Бэкенд         | Отсутствует                      |

**Ключевое решение:** нет бэкенда, нет аккаунтов. Весь контент — JSON-файлы внутри приложения. Прогресс — AsyncStorage на устройстве. Это упрощает разработку и устраняет риски с инфраструктурой.

**Разделение ответственности состояния:**
- **Zustand** — runtime-состояние текущей сессии: какое задание сейчас, счётчик ошибок в уроке, текущий ввод пользователя. Сбрасывается при закрытии урока.
- **AsyncStorage** — персистентный прогресс между сессиями: пройденные уроки, лучшие звёзды, количество попыток. Читается при старте приложения, обновляется при завершении урока.

## Архитектура контента

### Иерархия

```
Учебник (1 класс, 1956)
 └── Главы (по темам)
      └── Уроки
           └── Задания
```

### Структура JSON

```json
{
  "id": "ch1",
  "title": "Числа от 1 до 5",
  "lessons": [
    {
      "id": "ch1_l1",
      "title": "Урок 1. Число 1",
      "exercises": [
        {
          "type": "arithmetic",
          "question": "1 + 1 = ?",
          "answer": 2
        },
        {
          "type": "word_problem",
          "text": "На ветке сидела 1 птица. Прилетела ещё 1. Сколько птиц стало?",
          "answer": 2
        },
        {
          "type": "counting",
          "text": "Сколько яблок?",
          "icon": "🍎",
          "count": 3,
          "answer": 3
        }
      ]
    }
  ]
}
```

### Типы заданий

| Тип            | Описание                                              |
|----------------|-------------------------------------------------------|
| `arithmetic`   | Пример с пропуском: `2 + 3 = ?`                      |
| `word_problem` | Текстовая задача, числовой ответ                      |
| `counting`     | Посчитай предметы (иконки), числовой ответ            |

Все ответы — натуральные числа. Ввод через встроенную цифровую клавиатуру.

## Экраны и навигация

```
/                        → Главный экран (сетка глав)
/chapter/[id]            → Список уроков главы
/lesson/[id]             → Урок (задания последовательно)
/lesson/[id]/result      → Итог урока (звёзды)
/progress                → Общий прогресс
/settings                → Настройки (звук, размер шрифта)
```

### Главный экран
Сетка карточек глав. Каждая карточка:
- Название темы
- Прогресс-бар (уроков пройдено / всего)
- Статус: открыта / заблокирована (иконка замка)

### Экран урока
- Одно задание на весь экран, крупно
- Встроенная цифровая клавиатура (0–9, стереть, ОК) — системная не используется
- При правильном ответе: зелёная анимация ✓, переход к следующему заданию
- При ошибке: красное мигание поля, показ правильного ответа, затем переход дальше
- Ребёнок не застревает — ошибка не блокирует продвижение

### Итог урока
- Крупные звёзды (1–3)
- Кнопки: «Ещё раз» / «Дальше»

---

## Геймификация

### Звёзды за урок

| Ошибок    | Звёзд |
|-----------|-------|
| 0         | ⭐⭐⭐  |
| 1–2       | ⭐⭐   |
| 3+        | ⭐    |

- Урок считается пройденным при любом результате
- Можно перепройти урок и улучшить результат (сохраняется лучший)

### Разблокировка глав
- Следующая глава открывается когда все уроки текущей пройдены (минимум 1 звезда каждый)
- Линейная прогрессия — без ветвления

### Анимации
- Звёздочки "высыпаются" в конце урока
- Глава разблокируется с анимацией открытия замка
- Правильный ответ: краткая зелёная пульсация

## UX-принципы (дети 6–7 лет)

- Шрифт не менее 24sp
- Кнопки не менее 48×48dp с отступами
- Системная клавиатура не используется
- Весь интерфейс — на русском языке
- Никакого мелкого или сложного текста в интерфейсе
- Анимации короткие (не более 0.5с), не отвлекающие от задания

---

## Хранение прогресса (AsyncStorage)

```json
{
  "progress": {
    "ch1_l1": { "completed": true, "stars": 3, "attempts": 1 },
    "ch1_l2": { "completed": true, "stars": 2, "attempts": 2 },
    "ch1_l3": { "completed": false }
  }
}
```

При каждом завершении урока прогресс перезаписывается (сохраняется лучший результат по звёздам).

---

## Скоп MVP (1-й класс)

Планируемые главы по программе 1-го класса:
1. Числа от 1 до 5
2. Числа от 6 до 10
3. Сложение в пределах 10
4. Вычитание в пределах 10
5. Числа от 11 до 20
6. Сложение и вычитание в пределах 20

Контент вводится вручную из оригинального учебника 1956 года в JSON-файлы.

---

## Вне скопа MVP

- Профили нескольких детей
- Облачный прогресс / синхронизация
- Монетизация и платный контент
- Классы 2–4
- Генерация заданий (все задания статичны)
- Push-уведомления
- Родительский кабинет / статистика

---

## Риски

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Ввод контента учебника занимает много времени | Высокая | Начать с 1–2 глав, остальное добавлять итерационно |
| Сложность с публикацией в РуСторе | Средняя | EAS Build генерирует AAB, РуСтор принимает стандартный Android-пакет |
| UI для детей сложно сбалансировать | Средняя | Тестировать на реальных детях 6–7 лет с первых прототипов |
